### 执行流程（仅执行原创歌曲模式，6首/天）

```
18:00 启动
  │
  ├─ [0] 发送开工通知邮件
  │
  ├─ [阶段一] 数据抓取
  │   ├─ music_chart_monitor.py（6大榜单+4大热点平台→HTML邮件）
  │   └─ fetch_charts.py → today_input.json
  │
  ├─ [阶段二] 做歌（Claude Code CLI）
  │   ├─ STEP A：话题推演 → 10个故事框架
  │   ├─ STEP A.5：今日制作简报（6首人设分配）
  │   │   ├─ 分组A：2首（参考曲来自多榜热歌）
  │   │   ├─ 分组B：2首（参考曲来自5位音乐人对标成名曲）
  │   │   └─ 分组C：2首（参考曲来自抖音影视综热点）
  │   ├─ STEP B：第一批3首（分组A1+B1+C1）每批并行
  │   ├─ STEP C：第二批3首（分组A2+B2+C2）
  │   └─ STEP D：归档 → 更新歌曲存档
  │
  └─ [阶段三] 发送执行报告邮件
```

### 每首歌6个Phase

1. 搜索原曲+下载+智能截取（librosa RMS+Onset定位chorus，-5s取90s）
2. 写歌词（Claude+Gemini比稿，8维Scorecard评分）
3. 歌名生成（豆包+Claude）
4. Suno V5 cover-inspo — `upload_audio(参考曲90s)` → `inspo_generate(description, lyrics)`
5. Phase 5 质检（5.0→5.0b→5.1→5.2→5.3，必须完整执行）
6. 封面图（豆包Seedream×2 + Gemini×2 = 4张/首）

### 失败处理

单首失败→换词重试1次→仍失败→跳过补下一首

## 任务4：每日视频生成

- **plist**：`~/Library/LaunchAgents/com.xuejin.dailyvideo.plist`
- **脚本**：`~/Documents/claude/自动化/daily-video/daily_video.py`
- **日志**：`/tmp/dailyvideo.log`
- **输出**：`~/Documents/claude/输出文件/视频/YYYY-MM-DD/`
- **每日产出**：2首歌 × 2版本（横+竖）= 4个视频 + 抖音文案
- **话题去重**：`used_topics.json` 记录已用影视话题，避免重复

### 执行流程

```
16:00 启动
  │
  ├─ 拉取抖音实时热搜（douyin_trending.py）
  ├─ 加载 rhythm_pool.json + 排除已做歌曲
  ├─ 选取2首歌（按rhythm_score降序）
  │
  └─ 每首歌：
      ├─ Gemini研究歌曲背景 + 热搜关联（trending_hooks）
      ├─ Gemini + Claude 双模型情绪判断 → 选模版
      ├─ 选背景图（从图库按情绪分类随机选）
      ├─ 下载音频 → librosa截取副歌高潮段15-20s
      ├─ Gemini歌词精确对齐（失败→终止，不回退鼓点）
      ├─ 生成竖版(1080×1920) + 横版(1920×1080)
      ├─ 质检：歌词同步、无乱码、背景不遮歌词
      ├─ 生成抖音文案 + Tags（含热搜话题）
      └─ 记录完成ID + 已用话题

每首歌额外生成3张不同风格的Veo动态背景备选
```

### 2种视频类型

**动态背景版**（全自动）
- Veo/AI图片做动态背景 + 歌词叠加
- 背景动态不占主视觉，歌词是主角
- 每首歌生成3张不同动态背景备选

**热门影视版**（半自动，待优化）
- 抓热搜筛选影视话题 → 匹配歌曲 → 下载抖音片段 → 合成
- 话题去重：used_topics.json，已用话题隔天不重复
- 素材要求：人脸完整、无字幕、无水印
